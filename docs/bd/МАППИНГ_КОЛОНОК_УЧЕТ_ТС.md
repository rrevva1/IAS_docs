# Маппинг колонок «Учет ТС» (AG Grid) на таблицы БД

Колонки представления «Учет ТС» приведены в соответствие с полями Основного учёта и отображаются в следующем порядке.

## Порядок колонок в гриде

| № | Колонка в гриде | Источник в БД | Примечание |
|---|------------------|---------------|------------|
| 1 | Пользователь | **users**.full_name (через equipment.responsible_user_id) | Ответственный за ТС |
| 2 | Помещение | **locations**.name (через equipment.location_id) | Кабинет/локация |
| 3 | ЦП | **part_char_values** (тип части «ЦП», характеристика «Модель») | При наличии связи equipment ↔ part_char_values |
| 4 | ОЗУ | **part_char_values** (тип «ОЗУ», характеристика «Объём») | |
| 5 | Диск | **part_char_values** (тип «Накопитель») | |
| 6 | Системный блок | **equipment**.name | Наименование СБ/тип техники |
| 7 | Инв. № | **equipment**.inventory_number | Инвентарный номер |
| 8 | Монитор | **part_char_values** (тип «Монитор») | |
| 9 | Имя ПК | **part_char_values** (характеристика «Имя ПК» / hostname) | |
| 10 | IP адрес | **part_char_values** (характеристика «IP адрес») | |
| 11 | ОС | **part_char_values** (характеристика «ОС») | |
| 12 | ДР техника | **equipment**.description | Другая техника, примечания |

## Таблицы

- **equipment** — основная таблица учёта ТС (id, name, inventory_number, responsible_user_id, location_id, description, …).
- **users** — пользователи; связь по equipment.responsible_user_id → users.id.
- **locations** — помещения; связь по equipment.location_id → locations.id.
- **part_char_values** — значения характеристик (при наличии в схеме: id_arm или equipment_id, id_part, id_char, value_text); справочники **spr_parts**, **spr_chars** задают типы частей и названия характеристик.

Подробные правила загрузки из файла Основного учёта — в `docs/РЕГЛАМЕНТ_ПАРСИНГА_ОУ_ТЕСТ.md`.
