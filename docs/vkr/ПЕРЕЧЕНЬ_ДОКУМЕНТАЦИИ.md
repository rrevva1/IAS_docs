# Перечень документации проекта ИАС учёта технических средств

Документ составлен по состоянию репозитория. Два фокуса: (1) что идёт в ВКР по плану, в том числе первые две главы; (2) рабочие документы системы вынесены в отдельную папку — см. [рабочие_документы_системы/](../рабочие_документы_системы/README.md).

---

## 1. Источники плана ВКР

- **План ВКР (конкретный):** [ПланВКР.txt](ПланВКР.txt) — тема, введение, цель, описание предметной области, модель хранения, входные/выходные данные, действия системы и пользователя, план работы по главам.
- **Примерный план:** [План_ВКР(примерный).txt](План_ВКР(примерный).txt) — структура разделов (исследовательский, проектный, технологический).

**Структура плана работы (первые две главы):**

| Раздел | Содержание |
|--------|------------|
| **Глава 1** | Анализ проблемы и формирование требований к системе |
| 1.1 | Постановка проблемы учета (в контексте проекта — учёт ТС и Help Desk) |
| 1.2 | Анализ предметной области и бизнес-процессов |
| 1.3 | Анализ существующих решений и аналогичных продуктов |
| 1.4 | Сравнение подходов (в плане — к извлечению метаданных; для ИАС УТС — к учёту/автоматизации) |
| 1.5 | Формирование требований к прототипу системы |
| **Глава 2** | Архитектурное проектирование прототипа ИАС |
| 2.1 | Выбор и обоснование архитектурного стиля |
| 2.2 | Проектирование компонентной структуры системы |
| 2.3 | Проектирование логической модели базы данных |
| 2.4 | Проектирование пользовательского интерфейса (UI/UX) |

Целевой объём первых двух глав по ГОСТ: **порядка 50–60 стр.**

## 2. Соответствие разделов ВКР и имеющейся документации

### 2.1. Глава 1 — что есть полностью, чего нет

| Пункт плана | Статус | Источники в проекте | Примечание |
|-------------|--------|----------------------|------------|

| **1.2 Анализ предметной области и бизнес-процессов** | Есть | TZ.md (разд. 4), SYSTEM_DESCRIPTION.md (разд. 2–4), карточка актива.md | Объект автоматизации, участники, контуры (Help Desk, активы, вложения, ПО/лицензии, аудит). Можно оформить в 10–15 стр. |
| **1.3 Анализ существующих решений и аналогичных продуктов** | Нет | — | Отдельного документа или раздела нет. Требуется написать 5–10 стр.: обзор аналогов (Help Desk, учёт ТС, корпоративные ИАС). |
| **1.4 Сравнение подходов** | Нет | — | В плане — «извлечение метаданных»; для ИАС УТС уместно «сравнение подходов к учёту ТС / к автоматизации Help Desk». Отдельного раздела нет, 3–5 стр. |
| **1.5 Формирование требований к прототипу** | Есть | TZ.md (разд. 5), docs/sistema/ТРЕБОВАНИЯ_К_СИСТЕМЕ.md | Функциональные и нефункциональные требования, выдержка в ТРЕБОВАНИЯ_К_СИСТЕМЕ. Достаточно для 8–12 стр. |

## 4. Оценка объёма текста для глав 1–2 (по ГОСТ)

Ориентир: ~30 строк на страницу, ~60 знаков в строке → ~1800 знаков на страницу. Оценка в страницах — приблизительная.

| Материал | Оценка (стр.) |
|----------|----------------|
| TZ.md (разд. 2–5) + SYSTEM_DESCRIPTION + карточка актива (для гл. 1) | ~15–20 |
| docs/sistema/ТРЕБОВАНИЯ_К_СИСТЕМЕ (для п. 1.5) | ~5–7 |
| docs/bd/МОДЕЛЬ_БД + ОПИСАНИЕ_И_ПЛАН_ДЕЙСТВИЙ_БД (для гл. 2.3) | ~8–12 |
| SYSTEM_DESCRIPTION + docs/sistema/СПЕЦИФИКАЦИЯ_API (для 2.1, 2.2, 2.4) | ~5–8 |
| **Итого имеющегося при оформлении в текст ВКР** | **~33–47** |
| Не хватает (1.1, 1.3, 1.4, 2.1, 2.2, 2.4 в развёрнутом виде) | **~15–25** |

Для выхода на **50–60 стр.** по первым двум главам целесообразно дописать: постановку проблемы (1.1), обзор аналогов (1.3), сравнение подходов (1.4), обоснование архитектурного стиля и компонентную структуру (2.1–2.2), сводное описание UI/UX (2.4).
