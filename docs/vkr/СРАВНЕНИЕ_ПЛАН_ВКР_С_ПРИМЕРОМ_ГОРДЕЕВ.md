# Сравнение плана ВКР (ИАС учёта ТС) с примером ВКР Гордеева (090401_23И1510_Гордеев_МИ)

Документ зафиксирован перед переходом к реализации ВКР. Цель — сопоставить объём и ход работы по примеру ВКР с планом по теме ИАС учёта технических средств предприятия.

## 1. Пример ВКР Гордеева — сводка

**Файл:** `090401_23И1510_Гордеев_МИ.pdf`  
**Тема:** Разработка системы автономной верификации устройства встроенного самотестирования памяти (применение: разработка микропроцессоров).

### 1.1. Объём (по аннотации)

| Показатель | Значение |
|------------|----------|
| Объём работы | 79 страниц |
| Рисунки | 14 |
| Таблицы | 10 |
| Листинги программ | 7 |
| Приложение | 14 страниц |

### 1.2. Структура по главам

- **Введение** — актуальность, цель, задачи, практическая значимость (в т.ч. на англ.).
- **Глава 1. Объект, предмет, проблема и цель исследования**
  - 1.1 Анализ существующих методов и подходов в автономной верификации устройств
  - 1.2 Анализ предмета, объекта и проблемы исследования  
    - 1.2.1 Определение цели и задач  
    - 1.2.2 Предпроектное обследование объекта  
    - 1.2.3 Анализ информационных процессов в области верификации  
    - 1.2.4 Сравнительный анализ аналогов средств автоматизации  
    - 1.2.5 Постановка задачи на разработку
- **Глава 2. Организационное обеспечение проекта**
  - 2.1 Анализ вариантов решения проблемы исследования
  - 2.2 Инфраструктура информационных технологий проекта  
    - 2.2.1 Системное и прикладное ПО  
    - 2.2.2 Инструментальное ПО  
    - 2.2.3 Техническое обеспечение
  - 2.3 Планирование реализации проекта  
    - 2.3.1 Резюме проекта  
    - 2.3.2 Иерархическая структура работ (WBS)  
    - 2.3.3 Оценка стоимости проекта  
    - 2.3.4 Расписание проекта  
    - 2.3.5 Календарно-ресурсный план
- **Глава 3. Результаты проектирования информационной системы**
  - 3.1 Сценарии работы пользователей (варианты использования, формы документов)
  - 3.2 Информационное обеспечение (выбор и обоснование архитектуры, модели информационной базы)
  - 3.3 Системы классификации и кодирования
  - 3.4 Обеспечение информационной безопасности
- **Заключение** (рус. и англ.)
- **Список использованных источников**
- **Приложение** (в т.ч. итоговая презентация)

Дополнительно: аннотация (рус./англ.), список сокращений, оглавление.

### 1.3. Ход работы (логика глав)

1. **Глава 1** — исследовательская: проблема, предметная область, аналоги, постановка задачи и требований.
2. **Глава 2** — организационно-управленческая: варианты решений, ИТ-инфраструктура (ПО, инструменты, железо), планирование (WBS, стоимость, расписание, ресурсы).
3. **Глава 3** — проектно-реализационная: сценарии использования, информационное обеспечение (архитектура, модель данных), классификация/кодирование, ИБ; по тексту — описание реализации, сборки, модулей, выводы по результатам.

То есть: сначала анализ и требования, затем организация и планирование проекта, затем проектирование и результаты (включая реализацию и выводы).

## 2. План ВКР по теме ИАС учёта ТС — сводка

**Файл:** `План_ВКР_ИАС_учета_ТС.txt`  
**Тема:** Проектирование и разработка прототипа информационно-аналитической системы учёта технических средств предприятия.

### 2.1. Структура по главам (из плана)

- **Введение** — проблема разрозненного учёта активов и Help Desk, цель (новое качество учёта за счёт централизации и автоматизации), кратко предметная область и прототип.
- **Глава 1. Анализ проблемы и формирование требований к системе**
  - 1.1 Постановка проблемы учета ТС и процессов Help Desk  
  - 1.2 Анализ предметной области и бизнес-процессов  
  - 1.3 Анализ существующих решений и аналогичных продуктов  
  - 1.4 Сравнение подходов к учёту активов и автоматизации Help Desk  
  - 1.5 Формирование требований к прототипу системы
- **Глава 2. Архитектурное проектирование прототипа ИАС учета ТС**
  - 2.1 Выбор и обоснование архитектурного стиля  
  - 2.2 Проектирование компонентной структуры системы  
  - 2.3 Проектирование логической модели БД  
  - 2.4 Проектирование пользовательского интерфейса (UI/UX)
- **Глава 3. Реализация и тестирование прототипа ИАС учета ТС**
  - 3.1 Обоснование выбора технологий и средств реализации  
  - 3.2 Реализация ключевых модулей прототипа  
  - 3.3 Демонстрация работоспособности прототипа  
  - 3.4 Тестирование работоспособности прототипа

В плане также заданы: описание предметной области (карточка актива, карточка пользователя, заявка Help Desk, модель хранения, входные/выходные данные, действия системы и пользователя).

### 2.2. Ход работы по плану

1. **Глава 1** — исследовательская: проблема, предметная область, аналоги, подходы, требования (функциональные и нефункциональные).
2. **Глава 2** — проектная: архитектура, компоненты, модель БД, UI/UX (без отдельной главы по организации/планированию проекта).
3. **Глава 3** — технологическая: стек, реализация модулей, демонстрация, тестирование.

## 3. Сопоставление и выводы

### 3.1. Соответствие структуры

| Аспект | Гордеев | План ИАС учёта ТС |
|--------|---------|-------------------|
| Введение | Есть (актуальность, цель, задачи) | Есть (проблема, цель, предметная область) |
| Глава 1 | Объект, предмет, проблема, цель; анализ методов, объекта, процессов, аналогов; постановка задачи | Анализ проблемы, предметной области, аналогов, подходов; формирование требований |
| Глава 2 | Организационное обеспечение: варианты решений, ИТ-инфраструктура, планирование (WBS, стоимость, расписание) | Архитектурное проектирование: стиль, компоненты, БД, UI/UX |
| Глава 3 | Результаты проектирования ИС: сценарии, информационное обеспечение (архитектура, модель данных), классификация, ИБ | Реализация и тестирование: технологии, модули, демонстрация, тесты |
| Заключение / источники / приложение | Есть | Подразумеваются по общепринятой структуре ВКР |

Вывод: план ИАС учёта ТС по содержанию глав 1 и 3 хорошо стыкуется с примером (анализ → проектирование/реализация). Отличие: у Гордеева вторая глава — «организационное обеспечение и планирование проекта», у вас вторая глава — «архитектурное проектирование». То есть у вас проектный блок сосредоточен в гл. 2, реализация — в гл. 3; у Гордеева «организация проекта» вынесена в гл. 2, а «результаты проектирования» (включая архитектуру и реализацию) — в гл. 3.

### 3.2. Ориентиры по объёму

- **Объём примера:** 79 стр. основного текста, 14 стр. приложения, 14 рисунков, 10 таблиц, 7 листингов.
- **Рекомендация для ВКР по ИАС учёта ТС:** ориентироваться на сопоставимый объём (порядка 70–85 стр. без приложения), с учётом требований кафедры: достаточное количество рисунков (архитектура, модели данных, экраны), таблиц (сравнение аналогов, требования, тесты), при необходимости — фрагменты кода или конфигурации в виде листингов или в приложении.

### 3.3. Что учесть перед реализацией ВКР

1. **Глава 1** уже опирается на детальное описание предметной области в плане (карточка актива, карточка пользователя, заявка, модель хранения, входы/выходы, действия). При написании текста ВКР использовать это описание и при необходимости перенести в разделы 1.1–1.2; разделы 1.3–1.5 довести до явных требований и, при необходимости, таблиц.
2. **Глава 2** в вашем плане сильнее смещена в сторону архитектуры и проектирования (стиль, компоненты, БД, UI/UX). Этого достаточно для проектной части; при требовании кафедры к «организационному обеспечению» (планирование, сроки, ресурсы) можно добавить подраздел в гл. 2 или краткий раздел в гл. 3 (например, этапы и сроки реализации).
3. **Глава 3** по плану явно включает реализацию и тестирование — это соответствует «результатам проектирования и реализации» в примере Гордеева. Имеет смысл сохранить: обоснование стека (3.1), описание реализованных модулей (3.2), скриншоты и сценарии (3.3), описание тестов и результатов (3.4).
4. **Единый источник данных:** в плане и вводной части уже зафиксированы карточки активов, пользователей, заявки, импорт, отчётность, аудит. При реализации и при написании гл. 2–3 проверять соответствие прототипа этому описанию и явно отражать это в тексте (например, в разделах 3.2 и 3.3).
5. **Объём и оформление:** перед сдачей сверить объём и наличие разделов с примером Гордеева (введение, 3 главы, заключение, источники, приложение; при необходимости — аннотация, список сокращений, оглавление).

## 4. Краткая фиксация для этапа реализации

- **Пример (Гордеев):** 79 стр., 3 главы (объект/проблема/цель → организационное обеспечение → результаты проектирования ИС), заключение, источники, приложение.
- **Ваш план:** 3 главы (анализ и требования → архитектурное проектирование → реализация и тестирование); описание предметной области и целей задано.
- **Целевой объём:** ориентир 70–85 стр. основного текста, плюс приложение; рисунки, таблицы, при необходимости листинги — по мере необходимости для раскрытия темы и требований кафедры.
- **Перед реализацией:** зафиксировано соответствие плана примеру по структуре и объёму; при реализации придерживаться плана (модули из п. 3.2 плана, демонстрация из 3.3, тестирование из 3.4) и при написании текста ВКР опираться на описание предметной области и требований из плана.

Документ можно использовать как чеклист при написании глав и при приёмке объёма и структуры ВКР.
